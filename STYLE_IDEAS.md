# Portfolio style ideas – quick reference

Use this when we discuss theme, projects layout, and transitions. Pick one direction or mix elements (e.g. "A's colors with C's card grid").

---

## Option A – Soft dark tech

- **Background:** Dark (#0f1419–#161b22), not pure black
- **Accents:** Soft blue + cyan gradients, low saturation
- **Cards:** Rounded, subtle glow/border, gentle shadow
- **Vibe:** Professional dev portfolio, calm, current-GitHub-ish
- **Best for:** Keeping your existing "tech" feel but less heavy

---

## Option B – Warm minimal

- **Background:** Off-white / cream (#faf8f5, #f5f2ed)
- **Accents:** Warm (amber, terracotta, soft orange)
- **Cards:** Light background, thin border, lots of padding
- **Vibe:** Editorial, readable, friendly
- **Best for:** Standing out from typical dark dev portfolios; good for "personal" + "Beyond code"

---

## Option C – Ebi-style grid

- **Background:** Neutral dark gray
- **Layout:** Single grid, image-on-top cards, same card style for every project
- **Accents:** One main color (e.g. blue or green) for links and buttons
- **Cards:** Thumbnail on top → title → short description → tech tags → "Live" / "GitHub"
- **Vibe:** Clean, image-first, like ebi-manoj reference
- **Best for:** Projects section "like ebi-manoj"; clear hierarchy

---

## Option D – High contrast bold

- **Background:** Very dark (#0a0a0a)
- **Accents:** One strong color (e.g. electric blue or green #00ff88)
- **Typography:** Large, bold headings
- **Cards:** Strong border or outline, clear separation
- **Vibe:** Confident, terminal/CLI feel
- **Best for:** Maximum clarity and "dev" personality

---

## Mix-and-match checklist

When we implement, we can combine:

- [ ] **Colors:** A / B / C / D (or hybrid)
- [ ] **Projects layout:** C's uniform grid + image-on-top
- [ ] **Light mode:** Optional (fits B best; can add to A/C/D)
- [ ] **Transitions:** Lighter (fewer glows, respect reduced-motion) – works with any option

Tell me which option (or mix) you prefer and we'll align the plan and code to that.

## CHOSEN: Alex-style layout + techie vibe

You like the **Alex-style** reference (warm minimal, off-white/beige, terracotta accent, clean grid, image-on-top cards, category tags). We blend that with a **techie vibe** so it still feels like a dev/AI portfolio.

**Keep:** Light airy background (#faf8f5), simple header + nav, hero with one CTA, **uniform project grid** (image → title → description → tags), soft shadows, rounded corners, minimal footer.

**Add techie:** (1) **Accent** — blue/cyan/teal or soft green instead of terracotta (CTA, nav underline, tags). (2) **Typography** — monospace (JetBrains Mono) for "$ whoami", code labels, or tags. (3) **Subtle tech** — very light grid or gradient in hero; soft hover glow on cards/buttons. (4) **Tags** — same pill style, tech labels (Python, React, ML, AWS). (5) **Placeholders** — gradient or icon for projects without images.

Result: same calm minimal layout, clear dev/tech identity.
